## 🧠 오답노트: 프로그래머스 Lv.3 — 입국심사 (이분 탐색)

### 📅 날짜

2025.11.03

---

### 🧩 문제 요약

여러 심사관이 있을 때, **n명의 사람을 모두 검사하는 데 걸리는 최소 시간**을 구하는 문제.

- 각 심사관의 처리 시간 배열: `times`
- `n`: 총 인원 수
- 답: 모든 사람을 처리하는 데 걸리는 **최소 시간**

---

### 🚫 나의 실수 / 헷갈렸던 부분

| 구분           | 내용                            | 정리                                             |
| -------------- | ------------------------------- | ------------------------------------------------ |
| 탐색 대상 착각 | 사람 수를 탐색하려 함           | ✅ 시간(mid)을 탐색해야 함                       |
| check() 헷갈림 | count 계산 목적 불명확          | ✅ mid 시간 안에 처리 가능한 인원 수 계산용 함수 |
| 조건 반전      | `if (count > n)` 과 `>=` 헷갈림 | ✅ `>=` → 가능, `else` → 불가능                  |
| while 조건     | `left < right` 으로 작성        | ✅ `left <= right` 로 해야 마지막 후보 포함      |

---

### ✅ 배운 점 요약

1. **이분 탐색의 본질**

   - 답이 될 수 있는 **값의 범위(시간)** 를 반씩 줄여가며 탐색한다.
   - `mid`는 실제 값이 아니라 **가정(추측)** 이다.

2. **check(mid)** 의 역할

   - mid 시간 동안 각 심사관이 처리할 수 있는 인원 수의 총합 계산

   ```js
   const check = (mid, times) => {
     let total = 0;
     for (const t of times) {
       total += Math.floor(mid / t);
       if (total >= n) break; // 조기 종료
     }
     return total;
   };
   ```

3. **이분 탐색 로직의 패턴 (암기 필수)**

   ```js
   while (left <= right) {
     const mid = Math.floor((left + right) / 2);
     const count = check(mid, times);

     if (count >= n) {
       answer = mid; // 가능한 시간 후보 저장
       right = mid - 1; // 더 짧은 시간 시도
     } else {
       left = mid + 1; // 더 긴 시간 필요
     }
   }
   ```

4. **시간복잡도**

   - 이분 탐색: O(log(maxTime))
   - check 함수: O(times.length)
   - 전체: O(m log(maxTime))

---

### 💬 깨달은 점

> 이분 탐색은 **그래프 탐색이 아니라 값의 범위를 탐색**하는 알고리즘이다.
> `check()`는 mid가 답이 될 수 있는지 판별하는 **검사기 역할**이다.
>
> 처음엔 외워야 하지만, mid를 ‘가정’이라고 생각하면 구조가 감으로 잡힌다.
